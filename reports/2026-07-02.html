How to Use RLMs in Deep Agents

#langchain #deep_agents #subagent #harness_engineering #long_context #long_horizon #runtime #evaluation prototype strategic 신호 5/5
Deep Agents는 RLM 방식으로 에이전트가 코드를 작성해 dynamic subagents를 context chunk별로 dispatch하고, grep/map/reduce식 fan-out으로 long-context rot를 줄이는 하네스 접근입니다.
원문 상세 요약 - 원문은 Recursive Language Models가 context rot 문제를 줄이기 위해 agent가 직접 코드를 작성하고 subagent를 dispatch한다고 설명합니다. - Deep Agents는 dynamic subagents와 lightweight code interpreter로 이 방식을 구현한다고 되어 있습니다. - 큰 입력을 한 context window에 넣는 대신 context chunk별로 작업을 분해합니다. - 작업 패턴은 grep, map, reduce처럼 프로그램적으로 fan-out하는 구조입니다. - 평가에는 OOLONG이라는 long-context reasoning task가 사용됐습니다. - 원문은 turn-by-turn agent가 무너지기 시작하는 지점에서도 이 접근이 버틴다고 주장합니다. 인사이트 - 장기 작업 agent의 병목은 모델 context 길이만이 아니라 dispatch, 중간 산출물, merge, verification을 관리하는 runtime 설계입니다. - subagent framework와 code interpreter가 결합되면 agent harness는 단순 chat loop가 아니라 작은 실행 엔진에 가까워집니다. 확인 포인트 - Deep Agents 코드/패키지 공개 범위, OOLONG 재현 가능성, subagent 중간 결과 검증, retry, cancellation, cost accounting 정책을 확인해야 합니다.
