Fault Tolerance in LangGraph: Retries, Timeouts and Error Handlers

#langchain #langgraph #agent_framework #harness_engineering #fault_tolerance #workflow #orchestration released high 신호 4/5
LangGraph fault tolerance 글은 RetryPolicy, TimeoutPolicy, error_handler, SAGA 패턴을 production agent workflow engine의 기본 primitives로 제시합니다.
원문 상세 요약 - LangChain은 production agents가 prototype과 다른 방식으로 실패한다고 설명했습니다. - 글은 LangGraph에 내장된 세 가지 fault tolerance primitive를 다룹니다. - RetryPolicy는 backoff를 포함한 automatic retries를 담당합니다. - TimeoutPolicy는 wall-clock 기준과 idle 기준 cap을 제공합니다. - error_handler는 retries가 소진된 뒤 cleanup logic을 실행하는 데 쓰입니다. - LangChain은 이 기능들이 workflow engine 내부에 있어야 하는 이유와 서로 compose되는 방식을 설명한다고 밝혔습니다. - 다단계 workflow가 real-world side effects를 가질 때 SAGA pattern을 사용하는 방법도 다룹니다. 인사이트 - production agent의 신뢰성은 프롬프트 개선보다 retry, timeout, compensation, cleanup을 workflow engine에 내장하는 쪽에 가까워지고 있습니다. - 외부 API 호출, 파일 변경, 배포 작업을 수행하는 에이전트에는 SAGA식 보상 트랜잭션 설계가 중요합니다. 확인 포인트 - 각 policy의 SDK 예제, 기본값, observability 이벤트, 실패 재현 테스트 예제를 확인해야 합니다. - side effect rollback이 실제로 어디까지 지원되는지, 사용자가 직접 작성해야 하는 보상 로직 범위가 어디인지 확인해야 합니다.

Delta Channels: How We’re Evolving our Runtime for Long-Running Agents

#langchain #langgraph #agent_framework #harness_engineering #runtime #checkpointing #long_running_agents released high 신호 4/5
Delta Channels는 LangGraph 1.2의 새 primitive로, 장기 실행 에이전트의 checkpoint 저장 비용이 O(N²)로 커지는 문제를 diff 저장으로 줄입니다.
원문 상세 요약 - LangChain은 long-running agents에 storage problem이 있다고 설명했습니다. - 기존 방식은 매 step마다 full state를 checkpointing하면 저장량이 O(N²)로 증가합니다. - DeltaChannel은 LangGraph 1.2의 새 primitive로, 매 step에서는 diff만 checkpoint하고 주기적으로 full snapshot을 기록합니다. - 목표는 세션이 길어져도 storage costs를 flat하게 유지하는 것입니다. - 이 기능은 Deep Agents v0.6에 기본 탑재됩니다. - 설정 변경이나 데이터 마이그레이션 없이 제공된다고 설명됐습니다. 인사이트 - 장기 실행 에이전트의 병목은 컨텍스트 길이뿐 아니라 저장소 비용, 재개 가능성, checkpoint 설계입니다. - Delta checkpointing은 production agent runtime에서 비용과 복구성을 동시에 다루는 중요한 런타임 primitive입니다. 확인 포인트 - LangGraph 1.2 API에서 DeltaChannel 사용법, 호환되는 state type, snapshot 주기, 장애 복구 동작을 확인해야 합니다. - 실제 장기 세션에서 저장량과 복구 시간 개선 수치가 제공되는지 확인해야 합니다.
