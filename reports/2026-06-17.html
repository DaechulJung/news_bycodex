Recursive Agent Harnesses

#research #paper #harness_engineering #coding_agent #subagent #tool_use #benchmark #evaluation #long_horizon prototype strategic 신호 5/5
Recursive Agent Harnesses는 모델 호출 재귀가 아니라 파일시스템 도구, 코드 실행, 계획 기능을 포함한 전체 에이전트 하네스를 재귀 단위로 삼는 연구입니다.
원문 상세 요약 - 논문은 Recursive Language Models가 긴 문맥 추론에서 모델 호출 재귀를 활용한 흐름과, 프로덕션 코딩 에이전트가 하위 에이전트를 대규모로 생성하는 흐름 사이의 공통 패턴을 명명합니다. - 제안 개념은 Recursive Agent Harness, 즉 RAH이며, 단순 모델 호출이 아니라 filesystem tools, code execution, planning을 포함한 전체 agent harness를 재귀 단위로 사용합니다. - 부모 에이전트는 실행 가능한 스크립트를 생성하고 실행해 세분화된 작업을 병렬 subagent harness로 나누며, 작은 하위 작업에는 structured function calls를 사용합니다. - 평가는 long-context reasoning에 대한 controlled evaluation으로 제시됩니다. - 백본을 GPT-5로 고정해 Codex 및 RLM baseline과 맞춘 조건에서 Oolong-Synthetic 199개 샘플, 13개 context-length bucket, 최대 4M token 범위를 평가했다고 설명합니다. - 결과는 Codex coding-agent baseline 71.75%에서 RAH 81.36%로 상승했으며, 논문은 이 차이를 모델이 아니라 하네스 설계에 기인한다고 해석합니다. - 더 강한 백본인 Claude Sonnet 4.5를 사용할 경우 같은 설계가 89.77%에 도달했다고 보고합니다. 인사이트 - 이 논문은 장기 작업 성능을 모델 크기나 컨텍스트 길이만으로 설명하지 않고, 작업 분해·병렬 하위 하네스·구조화 호출을 포함한 하네스 아키텍처의 효과로 분리하려는 점이 중요합니다. 코딩 에이전트 평가에서는 모델명보다 하네스 구조, subagent 생성 정책, 파일시스템 권한, 실행 재현성이 핵심 변수가 됩니다. 확인 포인트 - Oolong-Synthetic 데이터셋과 199개 샘플 구성, 13개 bucket 분포, 평가 스크립트 공개 여부를 확인해야 합니다. - subagent 병렬 실행이 토큰·시간·비용을 얼마나 늘리는지, baseline과 동일 budget 조건인지 확인해야 합니다. - 파일시스템 도구와 코드 실행을 포함한 재귀 하네스가 실패하거나 무한 확장되는 경우의 stop condition과 safety gate가 필요합니다.

Orchestra-o1: Omnimodal Agent Orchestration

#research #paper #agent_framework #harness_engineering #orchestration #subagent #multimodal #benchmark #evaluation prototype strategic 신호 5/5
Orchestra-o1은 텍스트·이미지·오디오·비디오가 섞인 omnimodal 작업에서 modality-aware task decomposition, online sub-agent specialization, parallel execution을 결합한 agent orchestration 연구입니다.
원문 상세 요약 - 논문은 LLM 기반 에이전트 흐름이 single-agent workflow에서 multi-agent system으로 이동하면서 task decomposition과 collaboration을 위한 orchestration이 중요해졌다고 출발합니다. - 기존 orchestration framework는 좁은 modality에 묶여 있고, text, image, audio, video 같은 heterogeneous modalities가 함께 존재하는 복잡한 설정에서는 일반화가 어렵다고 지적합니다. - 제안 프레임워크 Orchestra-o1은 omnimodal agent orchestration을 목표로 하며, 여러 modality를 이해하고 조정하는 에이전트 협업을 지원합니다. - 핵심 메커니즘은 modality-aware task decomposition, online sub-agent specialization, parallel sub-task execution입니다. - 논문은 이러한 확장형 설계가 이질적 정보 소스를 포함하는 실제 복합 작업을 처리하도록 돕는다고 주장합니다. - 평가에서는 OmniGAIA benchmark에서 second-best approach 대비 accuracy 10.3%p 우위를 보고합니다. - 추가로 decision-aligned group relative policy optimization, DA-GRPO를 제안해 Orchestra-o1-8B를 훈련하며, 오픈소스 omnimodal agent 대비 state-of-the-art 성능을 달성했다고 설명합니다. 인사이트 - agent orchestration 연구가 단순 role assignment에서 modality-aware routing과 online specialization으로 이동하는 신호입니다. 하네스 관점에서는 입력 modality별 도구 선택, 병렬 하위 작업 추적, 결과 병합 기준, 멀티모달 평가셋 재현성이 핵심 확인 지점입니다. 확인 포인트 - OmniGAIA benchmark의 task 구성, train/test 분리, 평가 metric, second-best baseline 목록을 확인해야 합니다. - DA-GRPO가 실제 agent decision alignment를 얼마나 측정하는지, 단순 benchmark overfitting 가능성은 없는지 봐야 합니다. - 코드, 모델 가중치, orchestration runtime이 공개되는지 확인해야 재현 가능성을 판단할 수 있습니다.
