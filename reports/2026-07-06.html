원문 상세 요약 - 글은 Recursive Language Models, RLMs를 deep agents에 쓰는 방법을 다룹니다. - 핵심 문제는 큰 입력을 하나의 context window에 밀어 넣을 때 발생하는 context rot입니다. - 제안 방식은 agent가 context chunks를 대상으로 subagent를 dispatch하는 코드를 작성하게 하는 것입니다. - Deep Agents는 dynamic subagents와 lightweight code interpreter로 이를 구현한다고 설명됩니다. - agent가 grep, map, reduce와 유사한 fan-out 작업을 programmatically 수행할 수 있게 합니다. - OOLONG long-context reasoning task에서 벤치마크했으며, turn-by-turn agents가 무너지는 지점에서 더 버틴다고 소개됩니다. - 입력만으로는 OOLONG task 수, baseline 설정, 통계적 유의성, 코드 공개 여부는 확인되지 않습니다. 인사이트 - 장문 입력 처리를 단순 long-context 모델 의존에서 code-driven decomposition과 subagent orchestration으로 옮기는 중요한 harness pattern입니다. - agent가 자체적으로 map-reduce 형태의 작업 계획을 생성한다면 traceability, subtask scoring, partial result validation이 하네스의 핵심 평가 단위가 됩니다. 확인 포인트 - OOLONG benchmark의 데이터와 평가 스크립트가 재현 가능한지 확인해야 합니다. - lightweight code interpreter가 실행 격리, capability 제한, durable pause를 어떻게 제공하는지 확인해야 합니다. - dynamic subagent dispatch가 비용과 latency를 얼마나 증가시키는지 별도 측정이 필요합니다.
