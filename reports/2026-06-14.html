Recursive Agent Harnesses

원문 상세 요약 - 논문은 Recursive language models가 모델 호출을 반복해 long-context reasoning을 개선한 흐름과, production coding agent가 subagent를 대규모로 생성하는 흐름 사이의 공통 패턴을 정의합니다. - 저자들은 재귀 단위가 단순 모델 호출이 아니라 filesystem tools, code execution, planning을 갖춘 full agent harness일 때 이를 Recursive Agent Harness, 즉 RAH라고 부릅니다. - RAH는 모델 재귀의 code-first 확장으로 설명되며, parent agent가 실행 가능한 script를 생성하고 실행해 fine-grained workload용 subagent harness를 병렬로 띄우는 구조입니다. - 작은 하위 작업에는 structured function calls를 함께 사용한다고 설명합니다. - 평가는 long-context reasoning을 대상으로 controlled evaluation 형태로 제시됩니다. - backbone을 GPT-5로 고정해 Codex 및 RLM baseline과 맞춘 조건에서, Oolong-Synthetic 199 samples, 13 context-length buckets, 최대 4M tokens 설정을 사용했습니다. - RAH는 Codex coding-agent baseline을 71.75%에서 81.36%로 끌어올렸고, 저자들은 이 향상이 model이 아니라 harness 설계에서 온다고 해석합니다. - 더 강한 backbone인 Claude Sonnet 4.5에서는 같은 설계가 89.77%에 도달했다고 보고합니다. 인사이트 - 핵심 novelty는 subagent orchestration 자체보다, 재귀의 단위를 LLM call에서 실행 가능한 agent harness로 바꾼 점입니다. - 하네스 엔지니어링 관점에서는 parallel subagent spawning, filesystem 권한, code execution 경계, structured call fallback을 평가 가능한 설계 축으로 끌어낸 논문입니다. - 모델 고정 조건에서 성능 차이를 주장하므로, 모델 선택보다 하네스 구조가 long-context task 품질에 미치는 영향을 따로 측정하려는 시도로 볼 수 있습니다. 확인 포인트 - Oolong-Synthetic task가 실제 coding-agent 장기 작업을 얼마나 대표하는지 확인해야 합니다. - parent script가 subagent를 생성할 때 권한·작업 디렉터리·실패 복구·비용 상한을 어떻게 제한하는지 원문/코드 확인이 필요합니다. - 재현용 코드, prompt, harness 설정, baseline implementation 공개 여부를 확인해야 합니다.
