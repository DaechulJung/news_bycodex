Delta Channels: How We’re Evolving our Runtime for Long-Running Agents

#agent_framework #harness_engineering #runtime #state #memory #orchestration #release released high 신호 4/5
LangGraph 1.2의 DeltaChannel은 장기 실행 agent state checkpoint를 diff 중심으로 바꿔 O(N²) storage 증가 문제를 줄이는 runtime primitive입니다.
원문 상세 요약 - 원문은 long-running agents에 storage 문제가 있다고 설명합니다. - 매 step마다 full state를 checkpoint하면 storage가 O(N²)로 증가한다고 지적합니다. - DeltaChannel은 LangGraph 1.2의 새 primitive로 소개됩니다. - 매 step 전체 상태 대신 diff만 checkpoint합니다. - full snapshot은 주기적으로 기록하는 방식입니다. - 목표는 session이 길어져도 storage cost를 평탄하게 유지하는 것입니다. - Deep Agents v0.6에 기본 탑재되며 config 변경이나 data migration이 필요 없다고 설명합니다. 인사이트 - long-running agent의 병목이 token context뿐 아니라 checkpoint storage와 replay cost로 확장되고 있습니다. - 메모리와 상태 저장을 다루는 harness는 full snapshot, delta, compaction, replay semantics를 명확히 가져가야 합니다. 확인 포인트 - diff checkpoint의 충돌 처리, schema migration, rollback, replay 성능을 확인해야 합니다. - Deep Agents v0.6 기본 적용 범위와 LangGraph 1.2 호환성을 확인해야 합니다.
