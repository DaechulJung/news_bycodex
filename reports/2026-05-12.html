Delta Channels: How We’re Evolving our Runtime for Long-Running Agents

#langchain #langgraph #deep_agents #agent_runtime #checkpointing #harness_engineering #release released high 신호 4/5
LangGraph 1.2 DeltaChannel은 장시간 에이전트 체크포인트를 full state가 아닌 diff 중심으로 저장해 runtime storage 비용을 줄이는 공식 release 신호입니다.
원문 상세 요약 - LangChain은 long-running agents가 매 step마다 full state를 checkpoint하면 세션 길이가 길어질수록 O(N²) storage 문제가 생긴다고 설명합니다. - DeltaChannel은 각 step의 변경분만 저장하고 주기적으로 full snapshot을 기록하는 LangGraph 1.2 primitive입니다. - 목표는 long session에서도 checkpoint storage cost를 flat하게 유지하는 것입니다. - 입력 기준 Deep Agents v0.6에는 기본 적용되며 별도 config 변경이나 data migration 없이 제공됩니다. - 개발자 관점에서는 agent runtime의 persistence, recovery, long-session cost control에 직접 연결됩니다. 인사이트 - production agent harness에서 장시간 실행, 장애 복구, state audit은 비용과 안정성의 핵심 병목입니다. DeltaChannel은 모델 성능 발표가 아니라 agent runtime 운영면을 개선하는 release라 우선순위가 높습니다. 확인 포인트 - LangGraph 1.2 release notes, checkpoint backend별 동작, snapshot interval tuning, diff replay latency, rollback/compatibility behavior를 검증해야 합니다.

LiteParse Server: Self-Hostable Document Parsing

#llamaindex #liteparse #document_parsing #self_hosted #rag #observability #docker released medium 신호 3/5
LlamaIndex LiteParse Server는 문서 파싱을 self-hosted HTTP backend로 제공해 RAG·agent tool pipeline의 보안과 관찰성을 개선하는 tooling release입니다.
원문 상세 요약 - liteparse-server는 naive extraction tools와 expensive cloud APIs 사이를 겨냥한 self-hosted document parsing HTTP backend입니다. - 빠르고 local 실행 가능하며 정확한 parsing을 제공하는 것이 목표라고 설명됩니다. - raw text dump가 아니라 text의 spatial layout을 보존하며, 이는 table extraction과 citation grounding에 중요하다고 강조합니다. - 지원 형식은 PDF, Word documents, PowerPoints, spreadsheets, images입니다. - 주요 endpoint는 position data가 포함된 structured text 반환과 vision-capable AI model용 page image rendering입니다. - slim version은 infrastructure dependency 없이 실행되고, fuller setup은 caching, rate limiting, tracing, metrics를 추가합니다. - pre-built Docker image로 바로 실행할 수 있다고 입력에 제공되어 있습니다. 인사이트 - 문서 기반 agent는 parser 품질이 retrieval, grounding, citation accuracy를 좌우합니다. self-hosted parsing backend는 enterprise RAG harness에서 보안, 비용, throughput, observability를 직접 통제할 수 있게 합니다. 확인 포인트 - Docker image 경로, license, OCR 품질, table extraction 품질, tracing/metrics backend, large-file throughput, API compatibility를 fixture 기반으로 확인해야 합니다.
