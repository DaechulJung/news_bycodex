Recursive Agent Harnesses

#harness_engineering #subagent #coding_agent #benchmark #evaluation #research prototype high 신호 4/5
Recursive Agent Harnesses는 전체 agent harness를 재귀 단위로 삼고, Oolong-Synthetic에서 Codex baseline 71.75% 대비 81.36%를 보고한 subagent orchestration 연구입니다.
원문 상세 요약 - 논문은 Recursive language model의 model-call recursion과 production coding agent의 subagent spawning 패턴 사이를 연결해 Recursive Agent Harness(RAH)로 정리합니다. - RAH의 재귀 단위는 도구 없는 모델 호출이 아니라 filesystem tools, code execution, planning을 포함한 전체 agent harness입니다. - parent agent가 executable script를 생성·실행해 fine-grained workload용 subagent harness를 병렬로 띄우고, 작은 하위 작업에는 structured function call을 사용합니다. - 평가는 long-context reasoning을 대상으로 통제 실험 형태로 제시됩니다. - backbone을 GPT-5로 고정해 published Codex 및 RLM baseline과 맞췄고, Oolong-Synthetic 199 samples, 13 context-length buckets, 최대 4M tokens 조건을 사용했습니다. - 결과는 Codex coding-agent baseline 71.75%에서 RAH 81.36%로 상승했으며, 논문은 이 이득을 model 교체가 아니라 harness 설계 효과로 해석합니다. - 더 강한 backbone인 Claude Sonnet 4.5에서는 같은 설계가 89.77%에 도달했다고 보고합니다. 인사이트 - developer 관점의 핵심은 subagent framework가 실제 long-context task에서 성능 차이를 만들 수 있다는 주장입니다. 다만 production 적용성은 orchestration 비용, 로그 추적성, 권한 경계, 실패 복구 설계가 좌우합니다. 확인 포인트 - 코드, eval artifacts, Oolong-Synthetic 데이터 공개 여부를 확인해야 합니다. - subagent 병렬화가 토큰 비용, wall-clock latency, flaky tool calls, sandbox isolation에 어떤 부담을 주는지 추가 검증이 필요합니다.
