How to Use RLMs in Deep Agents

LangChain은 Deep Agents에서 RLMs를 이용해 code interpreter가 subagent fan-out과 reduce 흐름을 구성하는 long-context 하네스 접근을 소개했습니다.
원문 상세 요약 - 입력에 따르면 RLMs는 모든 정보를 하나의 context window에 넣지 않고, 에이전트가 context chunk별 subagent dispatch 코드를 작성하게 하는 방식입니다. - Deep Agents는 dynamic subagents와 lightweight code interpreter로 이 접근을 구현합니다. - 에이전트는 grep, map, reduce 같은 형태로 큰 입력을 programmatic하게 fan-out 처리할 수 있습니다. - 목적은 long-context 작업에서 발생하는 context rot를 줄이는 것입니다. - 평가에는 OOLONG long-context reasoning task가 사용됐습니다. - 입력 요약은 turn-by-turn agents가 무너지기 시작하는 구간에서 이 접근이 버틴다고 설명합니다. 인사이트 - 개발자에게 중요한 점은 long-context 성능이 모델 window 크기만의 문제가 아니라 execution topology와 subagent dispatch API 문제라는 것입니다. 장문 코드베이스 분석, 대규모 로그 분석, 문서 묶음 평가 하네스에 바로 적용 가능한 설계 방향입니다. 확인 포인트 - OOLONG task 구성, baseline, subagent 수, prompt budget, retry budget을 확인해야 합니다. - code interpreter가 생성한 dispatch logic의 권한, sandbox, replay 가능성을 봐야 합니다. - 단일 long-context 호출 대비 비용·latency·failure recovery가 어떻게 달라지는지 확인해야 합니다.
