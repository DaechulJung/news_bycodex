How Deep Agents Run Untrusted Code Without a Sandbox

LangChain은 Deep Agents의 agent-written orchestration code를 remote sandbox 대신 WASM+QuickJS 기반 in-process interpreter로 제한 실행하는 experimental runtime을 공개했습니다.
원문 상세 요약 - 2026-06-30 LangChain 글은 dynamic subagents에서 agent가 짧은 orchestration script를 작성하고 code interpreter가 이를 실행하는 패턴을 다룹니다. - 핵심 요구사항은 execution isolation, capability isolation, durable pauses입니다. - 실행 격리는 WebAssembly를 사용하며 WASM은 별도 linear memory를 갖고 host-provided capability를 통해서만 외부와 상호작용합니다. - 코드 실행 엔진은 QuickJS입니다. 작고 ECMA-compliant JavaScript engine이며 WASM으로 compile해 trusted surface를 줄인다고 설명합니다. - capability isolation은 file read, network request, dependency install을 기본 제공하지 않고 subagent 호출 같은 기능을 harness가 좁은 contract의 bridge로 명시 제공하는 방식입니다. - prompt injection이 해결되지 않았다는 전제에서 Meta의 rule of two를 언급하며 sensitive data, untrusted content, external state change 조합 위험을 설명합니다. - durable pause는 QuickJS interpreter의 linear memory를 LangGraph state로 serialize하고 human approval 후 snapshot을 restore해 이어서 실행하는 방식입니다. - quickjs-rs와 langchain-quickjs 두 experimental package가 공개됐습니다. 인사이트 - agent가 코드를 써서 orchestration하는 구조에서는 code execution이 성능 기능이자 보안 경계입니다. full sandbox와 lightweight interpreter 사이에 capability-first runtime이라는 중간 설계가 등장했습니다. 확인 포인트 - WASM escape, host bridge 권한 검증, snapshot 크기, approval audit log, prompt injection 상황에서 rule-of-two가 실제로 enforce되는지 확인해야 합니다.
