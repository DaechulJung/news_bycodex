# Noise Patterns

Treat generic AI listicles, recycled launch summaries, SEO-only tool directories, unsourced rumors, and duplicate reposts as low signal.

Promote weak signals only when they have credible source context, unusual developer traction, or clear relevance to agentic workflows.

## Editorial QA Rules

- Do not treat an aggregator source name as a provider signal. `Google Search`/Google News RSS is a discovery channel; add `#google` only when the title or cleaned summary mentions Google/Gemini, or the source is an official Google property.
- Do not tag runtime model mentions as model/provider launches. If an agent harness says it runs on Claude, Grok, or Gemini, classify by the harness unless the title or main claim is a model/API/pricing update.
- Provider names alone should not make a story hot. OpenAI/Anthropic/Google company mentions need model, Codex, agent, API, pricing, or adoption evidence before core placement.
- Low-engagement community launches stay as weak signals unless corroborated by another source or strong traction.
- AI workstation, GPU/NPU, cluster, and deployment hardware stories are `tooling` unless the main claim is a new LLM/model release.
- Core single-source items from Papers with Code, GitHub search, YouTube, or community sources need extra review unless they have strong traction or a corroborating source.
